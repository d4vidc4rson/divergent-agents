# Generate Diverse — Methodology

Structured five-pass protocol for generating genuinely different approaches to a problem, grounded in Guilford's 1967 divergent thinking framework. Each pass exercises a different cognitive dimension. Do not skip or combine passes.

---

## Pass 1: Fluency

Generate 8-12 ideas that address genuinely different sub-problems.

### Before generating:

1. Identify the Grade C trap for this specific problem — what would "ten variations of the same thought" look like here?
2. Name the convergence gravity so you can escape it.

### Grading system for real fluency:

- **Grade A** — Completely different ideas addressing different problems
- **Grade B** — Different ideas addressing similar problems
- **Grade C** — Similar ideas addressing similar problems
- **Grade D** — Academically identical ideas

### How to generate:

1. Before each idea, name the sub-problem it solves — not the same sub-problem reworded, a different one.
2. Identify the different dimensions of the problem that could each be addressed independently.
3. If you catch yourself solving the same sub-problem twice, delete the duplicate and find a new sub-problem.
4. For each idea: one sentence naming the sub-problem, one sentence stating the idea. No elaboration yet.

### Self-check:

Grade each pair of adjacent ideas using the A/B/C/D system. If any pair scores below B, replace the weaker one. Minimum standard is B across the board. Target A.

---

## Pass 2: Flexibility

Fight domain-specific clustering by routing the problem through unrelated disciplines.

### Method:

1. For each idea from Pass 1, ask: what field outside this domain would find this problem interesting?
2. Generate 3-5 new ideas by routing the problem through unrelated disciplines.

### How to pick the transfer domain:

Choose fields that have solved a structurally similar problem using completely different mechanisms:

- **Behavior change problems** -> epidemiology, game design, addiction psychology
- **Reliability problems** -> ecology, aviation safety, constitutional law
- **Discovery/search problems** -> forensic accounting, archaeology, immunology
- **Efficiency problems** -> logistics, ant colony behavior, emergency medicine triage
- **Persuasion/communication problems** -> trial law, documentary filmmaking, hostage negotiation

If none fit, pick the field that would be most confused by the problem as stated. Their confusion is diagnostic — what they find absurd is probably an assumption worth questioning.

### Rules:

- Write each transfer idea in the language of the original domain, not the transfer domain.
- Add the best 3-5 transfer ideas to the running list.

---

## Pass 3: Originality

Evaluate novelty. This is the only pass where you judge.

### For each idea, answer:

"If I searched for this approach, would I find someone already doing it?"

- **Yes, many people** -> Kill it.
- **Yes, a few people** -> Flag it. Might be necessary but won't differentiate.
- **Maybe one or two** -> Keep. This is the frontier.
- **No one I can find** -> Keep and star. This is where value lives.

### Expert Threat Test:

Would a credentialed professional in this domain nod along comfortably? If yes, it's not original enough. Would they feel defensive or say "you can't do that"? That friction is the signal.

**Safety-critical domains note:** The expert should feel their assumptions challenged, not their patients/users/public endangered. Intellectual friction, not physical recklessness.

### Actions:

1. Mark the three most original ideas.
2. Mark the three least original.
3. Remove the least original unless they serve a necessary structural function.

---

## Pass 4: Feasibility

Assign feasibility tags without resolving the originality-feasibility tension. Present it.

### Tags:

- **Now** — Implementable with existing resources and knowledge
- **Soon** — Requires some new capability or investment but clearly achievable
- **Stretch** — Requires significant new thinking or infrastructure but not impossible
- **Moonshot** — May not work but would be transformative if it did

### Quality criteria:

- A good output has ideas across all four tags.
- If everything is "Now," you were too cautious.
- If everything is "Moonshot," you were too abstract.

### For each Stretch and Moonshot idea:

Add one sentence on what would need to be true for it to become feasible. Not a plan. A condition: "This works if X turns out to be true."

---

## Pass 5: Clarity

Compress each surviving idea to its essential form. Idea quality is independent of idea length.

### Rules:

- Maximum 2-3 sentences per idea.
- One sentence naming the problem.
- One sentence stating the idea.
- One optional sentence on why it's non-obvious.

### Self-check:

If an idea can't be explained in three sentences, it's either too vague (you haven't found the idea yet) or too complicated (you've buried it in implementation). Strip every idea to its nerve.

---

## Quality Criteria for Final Output

1. 8-12 ideas surviving all five passes.
2. Adjacent-pair fluency grade of B or higher across the board.
3. At least 3-5 ideas sourced from cross-domain transfer (Pass 2).
4. Clear spread across feasibility tags (Now/Soon/Stretch/Moonshot).
5. No idea longer than 3 sentences.
6. The obvious/default answer is named explicitly for contrast.

---

## Self-Check Before Delivering

1. Read the full list. Can you identify the Grade C trap you named in Pass 1? Do any surviving ideas fall into it? If so, replace them.
2. Does the list contain at least one idea that would make a domain expert defensive? If not, originality filtering was too weak.
3. Is there a mix of feasibility levels? If everything clusters at one level, push the outliers.
4. Could each idea be understood by someone outside the domain in under 10 seconds? If not, compress further.
5. Name the convergence point — the default answer every other AI would give. If any of your ideas are indistinguishable from it, they failed.
